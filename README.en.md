# Fake API

## Project Overview

Fake API is a lightweight mock API service tool designed for front-end and back-end separation development. It allows front-end developers to quickly create mock APIs through simple configuration before the back-end APIs are completed, enabling independent development and testing.

## Features

- **Front-end and back-end separation**：Front-end can develop independently without waiting for back-end APIs
- **Easy to use**：Based on Node.js environment, intuitive to use with low learning cost
- **Flexible expansion**：Supports returning different data according to request parameters to meet complex business scenarios
- **Request logging**：Automatically saves request records for easy debugging and tracing
- **Non-intrusive development**：Only need to modify the baseUrl of the front-end project to switch to mock APIs
- **Support sharing**：Can be deployed and shared with team members for common use

## Installation

1. Clone or download the project to your local machine
2. Enter the project root directory and execute `npm install` to install dependencies
3. Create a `config.js` configuration file using the `config-template.js` template
4. Execute `npm start` to start the service

## Configuration

### Configuration File Structure

The following parameters can be configured through the `config.js` file：

- `port`：Service port number, default is 3000
- `apiDir`：Directory for storing API data files, default is `api`
- `logDir`：Directory for storing request logs, default is `logs`
- `cors`：Whether to allow cross-domain requests, default is `true`

### Configuration Example

```javascript
module.exports = {
  port: 3000,
  apiDir: 'api',
  logDir: 'logs',
  cors: true
};
```

## Usage

### 1. Start the Service

```bash
npm start
```

After the service starts, the service address will be output in the console, default format is `http://localhost:3000`.

### 2. Create APIs

Directly access the API URL you want to mock, for example：

```
http://localhost:3000/api/users
http://localhost:3000/api/articles/1
```

### 3. Write Return Data

After accessing the API, the system will automatically generate corresponding JSON files in the configured `apiDir` directory, for example：

- `api/api/users.json`
- `api/api/articles/1.json`

Fill in the data you want to return in the generated JSON files, for example：

```json
{
  "code": 200,
  "message": "success",
  "data": [
    {
      "id": 1,
      "name": "Zhang San",
      "age": 25
    },
    {
      "id": 2,
      "name": "Li Si",
      "age": 30
    }
  ]
}
```

### 4. Access the API Again

After saving the JSON file, access the same API URL again to get the configured return data.

## Advanced Features

### Return Different Data Based on Request Parameters

Fake API supports returning different data according to request parameters (such as query parameters, request body). You can use placeholders in JSON file names, for example：

- `api/api/users/[id].json`：Matches `http://localhost:3000/api/users/1`, `http://localhost:3000/api/users/2`, etc.
- `api/api/users?name=[name].json`：Matches `http://localhost:3000/api/users?name=Zhang San`, `http://localhost:3000/api/users?name=Li Si`, etc.

### Support for Multiple Request Methods

Fake API supports multiple HTTP request methods such as GET, POST, PUT, DELETE. For different request methods, you can create different files, for example：

- `api/api/users.GET.json`：Handles GET requests
- `api/api/users.POST.json`：Handles POST requests
- `api/api/users.PUT.json`：Handles PUT requests
- `api/api/users.DELETE.json`：Handles DELETE requests

## FAQ

### Q：How to modify the service port？
A：Modify the `port` parameter in the `config.js` file.

### Q：How to view request logs？
A：Request logs are automatically saved in the configured `logDir` directory, generated by date.

### Q：Does it support HTTPS？
A：Currently, HTTPS is not supported, only HTTP is supported.

### Q：How to handle cross-domain requests？
A：Set `cors: true` in the `config.js` file to allow cross-domain requests.
